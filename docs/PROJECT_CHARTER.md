### ADR-002: Local Executable Orchestrator (Phase 2)
**Date:** 2026-06

**Decision:** Added a minimal, one-file local CLI (`cli/council_cli.py`) so users can run full council leaps against their own vault exports or raw notes on their machine.

**Rationale:** Handing the user the lever is higher leverage than generating more leaps centrally. The personal memory prosthetic (Screenpipe-to-Obsidian) becomes the live steering signal for the discovery engine at runtime.

**Status:** Complete. The two systems now have a practical, local runtime bridge.
